Day 10 - GitHub Workflow
Date: 19-02-2026
Objective:
To understand GitHub workflow and collaboration using remote repositories.
Introduction:
GitHub is a cloud-based platform used to host Git repositories. It enables developers to collaborate, review code, and manage projects efficiently.
GitHub plays an important role in DevOps as it integrates with CI/CD tools.
Local vs Remote Repository:
Local Repository:
Stored on personal system.
Remote Repository:
Hosted on platforms like GitHub.

GitHub Workflow Steps Practiced:
1. Create repository on GitHub
2. Clone repository using:
git clone repository\_url
3. Make changes locally
4. Add and commit changes:
git add .
git commit -m "message"



5. Push changes:
git push origin main

Pull Request (PR):



A Pull Request is used to:
- Propose changes
- Review code 
=Merge changes into main branch



Steps:
- Create new branch
- Push branch to GitHub
- Open Pull Request
- Review and merge

GitHub Features:
- Issues tracking
- Pull Requests
- Branch protection
- Actions (CI/CD automation)

Importance in DevOps:
- Enables team collaboration
- Integrates with CI/CD pipelines
- Maintains version history
- Supports automated testing and deployment

Conclusion:
Today I understood GitHub workflow including cloning, pushing, pull requests, and collaboration using remote repositories.
