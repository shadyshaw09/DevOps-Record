Day 04 - Linux Basics
Date: 13-02-2026

Objective: To understand basic Linux operating system concepts and commonly used Linux commands.

Introduction: Linux is an open-source operating system widely used in servers, cloud computing, and DevOps environments. Most DevOps tools run on Linux-based systems.

Linux follows a hierarchical file system structure starting from the root directory (/).

Important Linux Directories:



- Root directory  
home - User home directories  
etc - Configuration files  
var - Variable data files  
bin - Essential command binaries  
usr - User system resources  

Basic Linux Commands Practiced:



1. pwd  
- Displays present working directory  
2. ls  
- Lists files and directories  
3. cd  
- Changes directory  
4. mkdir  
- Creates new directory  
5. rm  
- Removes files or directories  
6. touch  
- Creates empty file 
7. cat  
- Displays file content  
8. clear  
- Clears terminal screen  
 File Permissions:



Linux uses three types of permissions:

- Read (r)

- Write (w)

- Execute (x)



Permissions apply to:

- Owner

- Group

- Others



Command used:

chmod

Importance in DevOps:



Linux knowledge is essential in DevOps because:

- Most servers run Linux

- Docker containers are Linux-based

- CI/CD pipelines operate on Linux environments

 Conclusion:



Today I practiced basic Linux commands and understood file system structure and permissions. Linux is an essential skill for DevOps engineers.
