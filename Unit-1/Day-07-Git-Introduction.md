Day 07 - Git Introduction

Date: 16-02-2026

Objective:
To understand version control systems and the basics of Git.
Introduction:

Git is a distributed version control system used to track changes in source code during software development.
It allows multiple developers to work on the same project without overwriting each other's work.
Git is one of the most important tools in DevOps because it helps manage code efficiently.

What is Version Control?
Version control is a system that records changes to files over time so that specific versions can be recalled later.



Benefits:

- Tracks history of changes
- Enables collaboration
- Allows rollback to previous versions
- Prevents data loss

Types of Version Control:



1. Centralized Version Control (CVCS)
2. Distributed Version Control (DVCS)

Git is a Distributed Version Control System.

Git Architecture:

Working Directory  

Staging Area  

Local Repository  
These three areas help manage changes before pushing to remote repositories.

Basic Git Commands Practiced:
git --version  

git config --global user.name  

git config --global user.email  

git init  

Importance in DevOps:



- Enables collaboration

- Integrates with CI/CD pipelines

- Maintains project history

- Supports branching and merging
