Day 08 - Git Commands

Date: 17-02-2026

Objective: To practice commonly used Git commands for managing source code.
Introduction: Git provides various commands to manage files, track changes, and collaborate with other developers. Today we practiced basic Git workflow commands.
Basic Git Workflow:
Working Directory → Staging Area → Local Repository → Remote Repository

Git Commands Practiced:



1. git init  
Initializes a new Git repository.



2. git status  
Shows the status of files in the repository.



3. git add filename  
Adds files to the staging area.



4. git add .  
Adds all files to the staging area.



5. git commit -m "message"  
Commits changes to the local repository.



6. git log  
Shows commit history.



7. git clone repository\_url  
Clones a remote repository.



8. git pull  
Fetches and merges changes from remote repository.



9. git push  
Pushes local commits to remote repository.

 Git Workflow Example Practiced:



- Created a new folder
- Initialized Git using git init
- Created a file
- Added file using git add
- Committed changes using git commit
- Pushed to GitHub

Importance in DevOps:



Git commands are essential for:

- Managing project code

- Team collaboration

- CI/CD integration

- Tracking development progress

Conclusion:

Today I practiced important Git commands and understood how changes move from working directory to remote repository.
